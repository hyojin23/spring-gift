# spring-gift

## 기능 목록 및 개선 계획

### 1. Category

#### 구조 개선
- [x] CategoryController의 비즈니스 로직을 CategoryService로 분리
- [x] 카테고리 조회 책임을 Service 계층으로 이동
- [x] CategoryResponse 변환 책임을 정적 팩토리로 일관되게 정리
- [x] 카테고리 수정 시 존재 여부 확인 로직을 Service로 이동
- [x] 생성/수정/삭제 메서드에 트랜잭션 경계 설정

#### 테스트
- [x] 카테고리 목록 조회 테스트
- [x] 카테고리 생성 성공 테스트
- [x] 카테고리 수정 성공 테스트
- [x] 존재하지 않는 카테고리 수정 실패 테스트
- [x] 카테고리 삭제 성공 테스트

### 2. Wish

#### 구조 개선
- [ ] WishController의 비즈니스 로직을 WishService로 분리
- [ ] 위시 조회/추가/삭제 책임을 Service 계층으로 이동
- [ ] 상품 조회 로직을 Service로 이동
- [ ] 위시 중복 확인 로직을 Service로 이동
- [ ] 위시 삭제 권한 검증 로직을 Service로 이동

#### 작동 개선
- [ ] 존재하지 않는 상품 요청 시 예외 처리 추가
- [ ] 존재하지 않는 위시 삭제 시 예외 처리 추가
- [ ] 다른 사용자의 위시 삭제 요청 시 권한 검증 추가
- [ ] GlobalExceptionHandler 기반 예외 응답 일관화

#### 테스트
- [ ] 위시 목록 조회 테스트
- [ ] 위시 추가 성공 테스트
- [ ] 중복 위시 추가 테스트
- [ ] 존재하지 않는 상품 위시 추가 실패 테스트
- [ ] 위시 삭제 성공 테스트
- [ ] 다른 사용자의 위시 삭제 실패 테스트
