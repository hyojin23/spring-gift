# 기능 명세서: Option 패키지 예외 처리 리팩토링

**Feature Branch**: `002-option-exception-refactor`  
**작성일**: 2026-05-17  
**상태**: 초안  
**입력**: "예외 처리 기능을 리팩토링 한다. category 패키지에 되어 있는 것처럼 option 패키지도 똑같은 구조로 리팩토링 해야 한다."

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 시나리오 1 - Option 도메인 예외의 표준화된 응답 (우선순위: P1)

Option 기능에서 도메인 예외가 발생할 때 시스템은 `category` 패키지와 같은 방식으로 일관된 HTTP 에러 응답을 반환해야 합니다.

**우선순위 이유**: Option API 소비자는 상품 옵션 조회, 생성, 수정, 삭제 실패 상황에서 예측 가능한 에러 응답을 받아야 하며, 예외 처리 방식의 일관성은 유지보수성과 디버깅 효율을 높입니다.

**독립적 테스트**: Option 도메인 예외를 트리거하고 표준화된 HTTP 상태와 JSON 에러 형식을 검증합니다.

**승인 시나리오**:

1. **Given** 존재하지 않는 상품의 옵션을 조회할 때, **When** 클라이언트가 요청하면, **Then** 응답은 HTTP 404와 함께 표준 에러 JSON을 반환합니다.
2. **Given** 존재하지 않는 옵션을 수정하거나 삭제할 때, **When** 클라이언트가 요청하면, **Then** 응답은 HTTP 404와 함께 표준 에러 JSON을 반환합니다.
3. **Given** 이미 존재하는 옵션명으로 옵션을 생성하거나 수정할 때, **When** 중복 검증에 실패하면, **Then** 응답은 HTTP 400과 함께 표준 에러 JSON을 반환합니다.
4. **Given** 옵션이 1개만 남은 상품에서 마지막 옵션을 삭제할 때, **When** 삭제 제한 규칙에 걸리면, **Then** 응답은 HTTP 400과 함께 표준 에러 JSON을 반환합니다.

---

### 사용자 시나리오 2 - Category와 동일한 패키지 구조 적용 (우선순위: P2)

Option 패키지의 예외 처리 구조를 `category` 패키지와 동일한 형태로 맞춰, 팀이 일관된 예외 작성 및 관리 패턴을 사용할 수 있도록 합니다.

**우선순위 이유**: 일관된 구조는 코드 이해도를 높이고, 새로운 예외를 추가할 때 발생하는 중복과 혼선을 줄입니다.

**독립적 테스트**: 코드 리뷰를 통해 Option 예외 클래스 위치, 명명 규칙, 핸들러 통합 방식이 category 패키지와 일치하는지 확인합니다.

**승인 시나리오**:

1. Option 도메인 예외 클래스가 `gift.option.exception` 패키지 아래에 위치합니다.
2. Option 예외는 중앙 집중식 exception 핸들러에 의해 처리됩니다.
3. Option 컨트롤러에는 개별 `@ExceptionHandler`가 남지 않습니다.

---

### 사용자 시나리오 3 - 기존 Option 기능 동작 유지 (우선순위: P3)

리팩토링 후에도 기존 Option 기능의 모든 정상 동작과 API 계약이 변경되지 않아야 합니다.

**우선순위 이유**: 기존 API 사용자와의 호환성을 유지하는 것은 필수입니다.

**독립적 테스트**: 현재 Option 관련 통합 테스트를 실행하여 회귀 여부를 확인합니다.

**승인 시나리오**:

1. 기존 Option API의 정상 응답 HTTP 상태와 응답 본문 구조가 변경되지 않습니다.
2. 옵션 생성, 수정, 삭제의 비즈니스 규칙이 리팩토링 전과 동일하게 유지됩니다.
3. 잘못된 요청에 대한 HTTP 상태 코드는 기존 의미를 유지하되, 응답 본문은 표준 에러 형식으로 통일됩니다.

---

### 엣지 케이스

- 존재하지 않는 상품의 옵션 목록을 조회하거나 옵션을 추가하려는 경우 404 응답을 반환합니다.
- 존재하지 않는 옵션을 수정하거나 삭제하려는 경우 404 응답을 반환합니다.
- 같은 상품 안에서 중복된 옵션명을 생성하거나 수정하려는 경우 400 응답을 반환합니다.
- 옵션이 1개인 상품에서 마지막 옵션을 삭제하려는 경우 400 응답을 반환합니다.
- 옵션명 검증 오류가 여러 개 발생하면 클라이언트가 이해할 수 있는 메시지로 표준 에러 응답을 반환합니다.
- 매핑되지 않은 런타임 예외는 표준 에러 형식으로 처리되어 HTTP 500을 반환합니다.

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: Option 도메인 예외를 `gift.option.exception` 패키지로 이동하거나 재구성합니다.
- **FR-002**: `OptionNotFoundException`, `DuplicateOptionNameException`, `OptionDeletionNotAllowedException` 등의 Option 예외 클래스를 작성합니다.
- **FR-003**: Option 예외를 중앙 집중식 exception 핸들러에서 일관된 JSON 응답으로 변환합니다.
- **FR-004**: Option 컨트롤러에서 `orElse(null)`, `ResponseEntity.notFound()`, 개별 `@ExceptionHandler` 기반 예외 처리 로직을 제거하고 예외 발생 흐름으로 통일합니다.
- **FR-005**: Option의 상품 존재 여부, 옵션 존재 여부, 옵션명 중복 여부, 마지막 옵션 삭제 제한 규칙을 서비스 계층 또는 도메인 규칙으로 분리합니다.
- **FR-006**: Option 예외 처리 리팩토링을 검증하는 단위 및 통합 테스트를 추가합니다.
- **FR-007**: Option API 문서 또는 테스트 계약에 새 에러 응답 형식을 명시합니다.

### 주요 엔티티

- **OptionNotFoundException**: 요청한 옵션을 찾을 수 없을 때 발생합니다.
- **OptionProductNotFoundException**: 옵션을 조회하거나 추가할 대상 상품을 찾을 수 없을 때 발생합니다.
- **DuplicateOptionNameException**: 같은 상품 안에서 이미 존재하는 옵션명으로 생성 또는 수정하려 할 때 발생합니다.
- **OptionDeletionNotAllowedException**: 상품에 남은 옵션이 1개뿐이라 삭제할 수 없을 때 발생합니다.
- **OptionException**: Option 도메인 예외의 공통 기본 유형입니다.
- **ErrorResponse**: 클라이언트에 반환되는 표준화된 에러 페이로드입니다.
- **GlobalExceptionHandler**: 예외를 HTTP 상태 및 `ErrorResponse`로 변환하는 중앙 처리 컴포넌트입니다.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 리팩토링 후 모든 기존 Option 관련 테스트와 전체 테스트가 통과합니다.
- **SC-002**: Option 도메인 예외가 모두 표준화된 JSON 에러 형식으로 응답합니다.
- **SC-003**: Option 컨트롤러에서 개별 예외 처리 로직이 제거되어 중앙 처리로 일관화됩니다.
- **SC-004**: Category와 Option 패키지 간 예외 처리 구조 일관성이 코드 리뷰로 확인됩니다.
- **SC-005**: Option 비즈니스 규칙의 실패 케이스가 단위 또는 통합 테스트로 검증됩니다.

## 가정사항

- Category 패키지의 예외 처리 구조가 이미 완성되어 있고, Option 패키지는 이를 동일하게 따릅니다.
- 중앙 집중식 exception 핸들러(예: global controller advice)가 이미 존재하거나 이번 작업에서 사용 가능합니다.
- Option 패키지의 현재 컨트롤러는 예외 처리 로직과 비즈니스 규칙 일부가 분산되어 있습니다.
- 기존 Option API 사용자를 위해 정상 응답 형식과 성공 HTTP 상태 코드는 변경되지 않습니다.
- Product 도메인의 예외 처리 리팩토링은 별도 feature로 처리하며, 이번 작업에서는 Option API에서 필요한 상품 미존재 응답만 다룹니다.

---

**명세서 주석**: Option 패키지 외의 다른 도메인 예외 처리 리팩토링(예: product, member, order)은 별도 feature로 처리합니다.
