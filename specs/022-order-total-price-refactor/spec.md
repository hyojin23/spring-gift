# 기능 명세서: Order 총액 계산 책임 분리 리팩토링

**Feature Branch**: `022-order-total-price-refactor`  
**작성일**: 2026-05-20  
**상태**: 초안  
**입력**: "OrderService 가격 계산 책임 분리"

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 시나리오 1 - 주문 총액 계산 의도 명확화 (우선순위: P1)

주문 생성 시 회원 포인트 차감 금액은 상품 가격과 주문 수량을 곱한 주문 총액입니다. `OrderService`는 이 계산식을 주문 생성 흐름에 직접 노출하지 않고 명확한 이름의 메서드로 분리해야 합니다.

**우선순위 이유**: `product.price * quantity` 계산은 단순하지만 주문 결제 금액이라는 의미가 있습니다. 이름 있는 메서드로 분리하면 주문 생성 흐름이 더 읽기 쉬워지고 후속 할인/배송비/쿠폰 정책이 생길 때 변경 지점이 명확해집니다.

**독립적 테스트**: 상품 가격 1,000원, 수량 2개 주문 시 회원 포인트가 2,000원 차감되는 기존 테스트가 통과하는지 검증합니다.

**승인 시나리오**:

1. **Given** 상품 가격이 1,000원이고 주문 수량이 2개일 때, **When** 주문을 생성하면, **Then** 회원 포인트는 2,000원 차감됩니다.
2. **Then** 주문 저장, 위시 cleanup, 알림 호출 흐름은 기존처럼 수행됩니다.

---

### 사용자 시나리오 2 - 주문 실패 흐름 유지 (우선순위: P1)

가격 계산 메서드를 분리하더라도 재고 부족, 포인트 부족, 옵션 미존재 흐름은 기존처럼 동작해야 합니다.

**우선순위 이유**: 이번 작업은 구조 개선이며 동작 변경이 아닙니다. 주문 생성 실패 정책이 바뀌면 안 됩니다.

**독립적 테스트**: 기존 `OrderServiceTest`의 옵션 미존재, 재고 부족, 포인트 부족 테스트가 그대로 통과하는지 검증합니다.

---

### 사용자 시나리오 3 - 계산 정책 범위 유지 (우선순위: P2)

이번 작업은 총액 계산식을 이름 있는 메서드로 분리하는 것에 한정합니다. 할인, 배송비, 쿠폰, 포인트 적립 정책은 도입하지 않습니다.

**우선순위 이유**: 아직 추가 결제 정책 요구사항이 없습니다. 지금은 현재 정책을 명확히 표현하는 리팩토링으로 제한해야 합니다.

**독립적 테스트**: 계산 결과가 기존 `product.price * quantity`와 동일한지 기존 포인트 차감 테스트로 검증합니다.

---

### 엣지 케이스

- 주문 총액 계산식은 기존처럼 `option.getProduct().getPrice() * quantity`입니다.
- `quantity` 검증은 기존 `OrderRequest`, `Option`, `Order` 도메인 검증을 유지합니다.
- 포인트 부족 예외 정책은 변경하지 않습니다.
- 위시 cleanup과 주문 알림 호출 순서는 변경하지 않습니다.
- overflow 방지나 `long` 타입 전환은 이번 작업 범위에 포함하지 않습니다.

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: `OrderService`의 주문 총액 계산을 별도 private method로 분리해야 합니다.
- **FR-002**: 메서드 이름은 주문 총액 계산 의도를 드러내야 합니다.
- **FR-003**: 총액 계산식은 기존 `상품 가격 * 주문 수량`과 동일해야 합니다.
- **FR-004**: 회원 포인트 차감 금액은 분리된 총액 계산 메서드 결과를 사용해야 합니다.
- **FR-005**: 주문 생성 성공 시 재고 차감, 포인트 차감, 주문 저장, 위시 cleanup, 알림 호출 흐름을 유지해야 합니다.
- **FR-006**: 주문 생성 실패 응답/예외 흐름은 변경하지 않아야 합니다.
- **FR-007**: 기존 `OrderServiceTest`를 유지하거나 계산 의도를 더 명확히 검증하도록 보강해야 합니다.

### 주요 엔티티

- **OrderService**: 주문 생성 flow를 담당하며 총액 계산을 명명된 private method로 분리합니다.
- **Option**: 주문 옵션과 상품 가격 정보를 제공합니다.
- **Product**: 가격 정보를 제공합니다.
- **Member**: 계산된 주문 총액만큼 포인트를 차감합니다.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: `OrderService`에 주문 총액 계산 private method가 존재합니다.
- **SC-002**: 주문 생성 시 회원 포인트는 `상품 가격 * 수량`만큼 차감됩니다.
- **SC-003**: 주문 생성 성공 flow의 재고 차감, 주문 저장, 위시 cleanup, 알림 호출 테스트가 통과합니다.
- **SC-004**: 주문 생성 실패 flow 테스트가 기존처럼 통과합니다.
- **SC-005**: `./gradlew test --tests *Order*`가 통과합니다.
