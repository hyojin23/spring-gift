# Spec 문서 안내

`specs/` 디렉터리는 spec-kit 기반 SDD(Spec Driven Development) 작업 기록을 기능/리팩터링 단위로 보관합니다.
각 하위 디렉터리는 하나의 변경 주제를 나타내며, 여러 문서가 함께 생성될 수 있습니다.

## 리뷰어가 우선 확인할 문서

코드 리뷰 시에는 모든 파일을 동일한 비중으로 볼 필요가 없습니다.
리뷰어는 보통 아래 문서를 우선 확인하면 됩니다.

```text
spec.md   # 변경 목적, 범위, 요구사항, 성공 기준
plan.md   # 구현 방향, 영향 범위, 기술적 결정
tasks.md  # 구현 체크리스트와 완료 여부
```

특히 변경 의도와 리뷰 범위를 빠르게 파악하려면 `spec.md`와 `plan.md`를 먼저 보면 됩니다.
`tasks.md`는 실제 구현이 계획한 항목을 충족했는지 확인할 때 참고합니다.

## AI 및 도구 보조용 문서

아래 문서는 주로 AI가 구현을 진행하거나 검증 절차를 재현하기 위한 보조 문서입니다.
리뷰어가 필요할 때 참고할 수는 있지만, 기본 리뷰 대상은 아닙니다.

```text
research.md       # 구현 전 조사 내용과 판단 근거
data-model.md     # 관련 모델, 데이터 구조, 관계 정리
quickstart.md     # 구현 결과 확인 또는 실행 절차
contracts/        # API 응답, 계약, 인터페이스 초안
checklists/       # spec-kit 요구사항 점검 목록
```

## 디렉터리 읽는 순서

1. 변경 주제에 해당하는 `specs/{번호}-{주제}/` 디렉터리를 찾습니다.
2. `spec.md`에서 변경 목적과 요구사항을 확인합니다.
3. `plan.md`에서 구현 방향과 영향 범위를 확인합니다.
4. 필요하면 `tasks.md`로 완료된 작업 목록을 확인합니다.
5. 세부 판단 근거가 필요할 때만 `research.md`, `data-model.md`, `quickstart.md`, `contracts/`를 참고합니다.
